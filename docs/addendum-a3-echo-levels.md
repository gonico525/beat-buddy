# beat-buddy 追加仕様 A3「リズム型: レベル再編・タタ複数型の追加」 v1

> 既存実装（requirements.md v1.0 / addendum A1・A2）への**差分指示**。対象は**リズム型のレベル構成と型集合**。
> 採点方式（A2-4 の形採点）・テンポ規則（A1-4）・解禁の運用（保護者判断のみ, A1-8/§8）は変更しない。

---

## A3-0. 背景

A2-5 の末尾休符除外により、こだまの出題プールがグループ単位で極端に痩せた
（「タタタタ」= 1型、「おやすみ2」= 1型など）。同じ型ばかり出て飽きが早い。
グループを統合してプールを太らせ、型のバリエーションも増やす。

## A3-1. 解禁グループを2レベルに統合

- 旧6グループ（A1-8: `pattern_gap_0..3` / `pattern_split_1..2`）を廃止し、2レベルに再編する:
  - **`pattern_quarter`「タン と おやすみ」** = 4分のみの全8型（旧 L1〜L4）
  - **`pattern_split`「タタ いり」** = 細分入りの全型（旧 L5〜L6 + A3-3 の追加型）
- 解禁は従来どおり**保護者判断のみ・レベル単位**。自動昇格・降格なし、ロック中も可視（A1-8 を踏襲）。

## A3-2. タ・タ・（hrhr）をこだま出題に例外追加

- 末尾休符の除外（A2-5）は原則維持するが、**`hrhr` のみ例外**として出題する。
- 根拠: 打点2つの型の形採点（A2-4）は1打目と最終打点で決まり、末尾休符の有無は測定に影響しない
  （`hrrh` タ・・タ と同じ測定条件）。一方、呼びかけの聞こえ方（間の長さ）は異なるため出題価値がある。
- ほかの末尾休符型（`hhhr` `hhrr` `hrrr` `shhr` `hshr`）は引き続き除外。

## A3-3. タタ（8分細分）が2回入る型を追加

- split 2つ・休符なしの**全網羅 6型**を追加する:
  `sshh` `shsh` `shhs` `hssh` `hshs` `hhss`（すべて `pattern_split` レベル）。
- テンポ規則は変更なし: 細分を含む型の基準 IBI = SMT × 1.6（A1-4）。
- 型集合は全22型、こだまの出題プールは **17型・2レベル**（4分系5型 / タタ系12型）になる。

## A3-4. 既存プロフィールの引き継ぎ

- 保存済みの旧解禁キーは読み込み時に新レベルへ読み替える:
  `pattern_gap_0..3` → `pattern_quarter` / `pattern_split_1..2` → `pattern_split`。
- 旧キーのどれか1つでも解禁済みなら、対応する新レベルを解禁扱いとする。再ロックは新キー単位で行う。

## A3-5. 出題の重複回避

- こだまのリズム型出題は、**直前と同じ型を続けて出さない**（プールが2型以上のとき）。
- ふつうモード（発展）のバリエーション選択も同様に直前と同じものを続けない。
- 乱択自体は維持する（固定順にはしない）。

## A3-6. 影響範囲

- 採点（A2-4 形採点・A1-6 型採点）・出題フェーズの提示・応答フェーズの挙動は変更しない。
- 同期・知覚・全身の各層は変更しない。
