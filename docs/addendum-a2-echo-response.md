# beat-buddy 追加仕様 A2「こだま応答: 音だけ・任意開始・形の採点」 v1

> 既存実装（requirements.md v1.0 / addendum A1）への**差分指示**。対象は**こだま層の回答フェーズのみ**。
> 出題フェーズ・同期層など「パルスに合わせて叩く」他の層は変更しない。

---

## A2-0. 背景

現状、回答フェーズ（「きみのばん！」）で、叩く位置が緑の点付きグリッドとオノマトペ文字で表示されており、音を聞かなくても目で見て叩けてしまう。こだまは「聞いて覚えて返す」課題なので目的が損なわれている。またこの表示は細分パターンでのみ出ており、易しい4分のみパターンにはない——難易度と補助が逆転している。

## A2-1. 回答フェーズの表示を削除

- 4マスのグリッド表示、緑のドット（打点位置）、オノマトペ文字（例「タン タタ タン タン」）を回答フェーズでは**一切表示しない**。
- 画面は「きみのばん！」の見出しと太鼓のみ。
- 太鼓を叩いたときの反応（へこむ・光る・タップ音）は残す。これは**自分の動作へのフィードバック**であり、答えのヒントではない。
- 出題フェーズ（「きいてね」＝アプリが鳴らす番）の表示は現状のまま変更しない。音と同時に光ってよい。
- **全パターン共通の見た目**にする。4分のみと細分でUIを変えない。

## A2-2. 回答フェーズではパルスを鳴らさない

- 回答中はメトロノーム／パルス音を鳴らさない。**視覚パルスも出さない**。
- 理由: 拍の位置を視覚で示すと、子どもが耳ではなく光を追う。**聴覚→運動**の課題として成立させる。

## A2-3. 開始タイミングは子ども任意

- 「きみのばん！」表示後、子どもが**好きなタイミングで1打目を叩いてよい**。
- 特定の拍に入る（エントレイン）ことを要求しない。1打目を外すと以降全部ずれる、絶対時刻ベースの判定（A1-2 のターゲット照合を応答小節に適用する方式）はこだま応答では**使わない**。
  - A1-2 のエンジン照合そのものは「拍に合わせて型を叩く」系モード（A1-9 発展）用に温存する。

## A2-4. 採点を「絶対時刻の一致」から「形（相対比）の一致」に変更

- 期待パターンを、1打目を 0 とした相対時刻の列 `e[0..n]` で表す（局所期待間隔つき。8分は IBI/2, A1-6 踏襲）。
- 子の打点列 `t[0..n]` を取得し、`t[0]` を基準に相対化する: `d[i] = t[i] − t[0]`。
- スケール `s`（子が実際に叩いたテンポ / 目標テンポ）を推定する。簡易には**スパン比** `s = d[n] / e[n]` でよい。
- 各打点の相対誤差: `r[i] = |d[i] − s·e[i]| / (s × その打点の局所期待間隔)`。
- 判定は既存の段階を流用: `r ≤ 0.15` ぴったり ／ `≤ 0.30` いいね ／ それ以上は反応なし。
- **s（テンポ）が目標と違ってもよい。形が合っていれば正解**とする。ただし極端な場合のみ無効扱い（`s` が目標の **0.5倍未満／2.0倍超**）。この場合も失敗表示はしない（記録のみ）。
- 打点数が期待と違う場合は「形がちがった」として**記録のみ**。減点・失敗表示・降格はしない。
- 既存の非対称（当たりは出す／外れは罰しない）と、自動降格なし・自動昇格なしは維持する。

## A2-5. 末尾が休符のパターンをこだま層から除外

- 例「タタタ・」のように最後が休符の型は、子の最終打点で終端が定義できず、相対評価では測定不能。**こだま層の出題対象から外す**。
- 該当パターンは出題プールから除外し、除外後に空になるグループ（L4 タ・・・）は**こだまの選択肢・解禁UIにも表示しない**。解禁キー自体は温存する（既存プロフィールの値は無害。将来こだま以外の層で扱う場合に再利用可）。
- 除外対象: `hhhr` `hhrr` `hrhr` `hrrr` `shhr` `hshr`（末尾 rest の6型）。こだまの出題プールは**10型・5グループ**になる。

## A2-6. 影響範囲

- 出題フェーズ（きいてね）のロジック・表示は変更しない。
- 同期層など、パルスに合わせて叩く他の層は変更しない。今回の変更は**こだま層の回答フェーズのみ**。
- デバッグモードで、推定スケール `s` と各打点の相対誤差 `r[i]` を表示できるようにする。
