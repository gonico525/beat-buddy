# リズム感育成ウェブアプリ「beat-buddy」要件定義 v1.0

> Claude Code 引き渡し用ドラフト。値は根拠から引いた既定値。末尾「§12 チューニング可能パラメータ」は試遊後に調整する前提。

## 1. 目的とスコープ

- 幼児の**リズム感**を、知覚 → 同期 → 再現の順に育てる。対象年齢は固定しない（子の現在の層に合わせる）。
- 発達的事実に基づく設計: 同期は4〜4.5歳ごろに立ち上がり思春期まで発達が続く長い軌跡。幼児は自分の好みのテンポ（SMT, ITI≈400〜500ms）付近でしか同期できない。加速より減速が難しい。社会的文脈（親子）が同期を後押しする。

## 2. 非目標（作らないこと）

- マイクによる手拍子/足踏み検出（遅延・誤検出の沼、かつ採点しない層のため不要）。
- アカウント／サーバ／クラウド同期。永続化はローカルのみ。
- 採点は**同期・こだまの2層のみ**。知覚・全身は採点しない。
- 絶対的な入力遅延の厳密計測（ループバック等）。デバイス較正は近似で足りる（§5.1 の限界参照）。
- **パフォーマンス連動／時間経過による自動解禁・自動昇格**（§8 参照）。

## 3. 用語と符号約束

- **async（非同期）**: タップ時刻 − 可聴の拍時刻（秒）。**負＝先取り（早い・熟達サイン）／正＝遅れ**。全レイヤー共通の生値。
- **IBI**: 拍間隔（秒）。**IOI**: パターン内の音間隔。
- **SMT**: 子の自発運動テンポ（較正で測る）。
- **deviceOffset**: 端末の入力遅延を補正する1値（デバイス較正で算出）。
- **層(layer)**: 知覚 / 全身 / 同期 / こだま。手動選択・自動降格なし・自動昇格なし。

## 4. エンジン層（測定要件）

`rhythm-engine.ts` が担う。全採点はこの生 async の上に乗る純方針レイヤー。

- マスタークロックは `AudioContext.currentTime`。`Date.now`/`setTimeout` をタイミング源にしない。
- two-clock スケジューラ（粗い `setInterval` が「何を」、AudioContext が「いつ」）。音は合成（Oscillator）、外部アセットなし。
- 3座標系の整合: (1) `getOutputTimestamp()` でタップ(performance系)→AudioContext系, (2) `outputLatency` で可聴時刻補正, (3) `deviceOffset` で入力遅延補正。
- `onTap` は**生 async を配るだけ**で良否判定しない。採点レイヤーと検証用デバッグ表示を同じストリームに並列接続できること。
- フォールバック: `outputLatency` 無→`baseLatency`、`getOutputTimestamp` 無→start時取得のオフセット。

## 5. 較正要件（2種・別物）

### 5.1 デバイス較正（保護者・端末ごとに1回、再実行可）

- 手順: 一定拍（100bpm, IBI=600ms）を **16拍**提示。保護者は「音が鳴った瞬間に」タップ。
- 破棄: 最初の2拍。
- 算出: 各タップの生async（deviceOffset未適用）の**中央値** → `deviceOffset`。
- **既知の限界**: この値は端末の入力遅延＋保護者固有の先取り傾向の合算で、入力遅延単独ではない。系統ずれの中心化には十分と割り切る（マイク・ループバックは非目標）。
- 保存: `deviceOffsetMs`（ms）1値。

### 5.2 SMT較正（子ごと、プロフィールに保存／再測定可）

- 収集: 「すきに叩いてね」。最大 **12タップ or 8秒**、早い方で終了。
- 破棄: 最初の1タップ。
- デバウンス: **ITI < 120ms** は誤ダブルタップとして無視。
- 代表値: ITIの**中央値**。
- クランプ: **300〜700ms**（約86〜200bpm）に制限。範囲外は再測定を促す。
- 収束: 直近5 ITI の変動係数 CV < 0.20 で確定。未達なら上限まで継続、上限到達時は集めた中央値で確定。
- 出力: 初期テンポ = 中央値のbpm。同期・こだまの既定テンポ帯の基準。

## 6. 各層の受け入れ条件

各層: 入力／提示／終了条件／記録値／失敗時／相手（親子・ソロ）。**どの層も失敗で落とさない・自動降格なし**。発展モードの利用可否は §8 の親解禁に従う。

### 6.1 知覚（規則/不規則の当てっこ）※採点しない扱い（低ステークス）

- 提示: 2つの拍列を鳴らし「規則正しい方」を選ぶ2択強制選択。一方は一定、他方は位相/テンポ摂動。
- 試行数: 1セッション **4問**。
- 正解: 一定の方を選択。
- 記録: 正答率（表示は当てっこの体裁、失敗表示なし）。
- 相手: ソロ可。／解禁: **初期開放**。

### 6.2 全身（手拍子・足踏み）※採点しない

- 提示: 一定拍でキャラが手拍子/足踏み、子は身体で参加（画面タップ採点なし）。
- 終了条件: 提示拍（例 16拍）を流し切る＝**完了**。参加の質は測らない。
- 失敗概念なし。
- 相手: ソロ可（親子だとなお良い・任意）。／解禁: **初期開放**。

### 6.3 同期（合わせてタップ）※採点する

- 提示: SMT付近の一定拍。まず好みのテンポ帯のみ。
- 入力: `pointerdown` の `event.timeStamp` → エンジンへ。
- 採点: §7 の純関数。**当たりは出す／外れは罰しない**非対称。
- テンポ課題（発展・親解禁）: 難易度順 **速める→保つ→遅らせる**（減速は最後）。SMT基準の±%で段階化。
- 継続(continuation)（発展・親解禁）: 数拍提示後に**消音**し、子は「続けて叩く」。消音後タップITIの中央値と直前目標IBIのドリフト率を記録。ドリフト大でも記録のみ、小でポジティブ。
- 記録: 生async, 相対誤差 r, 段階ラベル（継続はドリフト率）。
- 相手: **親子を主役**。ソロ経路可（アプリがSMTに適応する前提。固定メトロノーム単独は避ける）。／解禁: 基本=親解禁、テンポ課題・継続=親解禁。

### 6.4 こだま（再現・産出）※採点する

- 提示: **2〜3音**の短パターン。IOIはSMT帯、単純な長さ（均等 or 1つ倍）。開始は2音・均等。
- 応答: 子が真似して返す。
- 一致判定: (a) オンセット数の一致、(b) 各IOIの相対誤差 ≤ 0.30。
- 記録: オンセット数, IOI誤差列。
- 相手: **親子主役**（呼びかけ↔応答は養育者交互性が土台）。／解禁: 基本=親解禁、3音/長短=親解禁。

## 7. 採点純関数（同期）

- 入力: `asynchrony`（秒）, 現 `IBI`。
- 相対誤差 `r = |async| / IBI`（テンポに自動スケール）。
- 段階:
  - `r ≤ 0.15` → 「ぴったり」（最上位ポジティブ）
  - `0.15 < r ≤ 0.30` → 「いいね」（ポジティブ）
  - `r > 0.30` → **反応なし（減点・降格なし）**
- 非対称: 外れは一切罰しない。連続ヒットで褒めを強めてよいが、連続ミスで失敗表示・降格はしない。
- こだま・継続も同系の純関数（相対誤差／ドリフト率）で表現する。

## 8. 層・機能の解禁（親判断）

- **原則**: 解禁は**親判断のみ**。パフォーマンス連動の自動解禁も、時間経過による自動解禁も行わない。リズム感は短期で伸びるものではなく思春期まで続く長い軌跡のため、短期の出来を昇格の合図にしない。自動降格なしと**対称に、自動昇格もなし**（進むも戻るも親の手）。
- **表示**: ロック中の項目も選択肢に**見える形で残す**（隠さず disabled 表示）。到達すべき梯子が親子に見えることを優先。
- **初期状態**: **知覚・全身のみ開放**、他はすべてロック。以降は親が任意に切り替える。
- **任意ロック/解除**: 親は**任意の項目を、任意の順で、個別に**ロック/解除できる。順序の強制・前提条件なし。梯子は**推奨表示順**としてのみ残す（下記）。ロック項目に到達必須の前段はない。
  1. 知覚 … 初期開放
  2. 全身 … 初期開放
  3. 同期(基本)
  4. こだま(基本)
  5. 同期: テンポ課題
  6. 同期: 継続
  7. こだま(3音/長短)
- **親ゲート**: 幼児が誤って解禁できないよう、解禁操作に大人ゲートを課す（例: 長押し＋一段階の算数確認）。プレリーダーの子でも親が実施可能。機構は §12 でチューニング可。
- **再ロック**: 親判断でいつでも・任意の項目に対して可。子操作では不可。
- **永続化**: 子プロフィールに `unlockedFeatures`（§10）。

## 9. 親子/ソロ経路

- **同期・こだま = 親子を主役**（一緒に叩く・呼びかけ↔応答のUI）。ソロ経路も提供。
- **知覚・全身・継続 = ソロ可**。
- 経路はモード切替として明示。ソロでも根拠と矛盾しない層のみソロ既定。

## 10. 永続化（localStorage）

- `device`: `{ deviceOffsetMs }`
- `children[]`: `{ id, name?, smtMs, reachedLayer, unlockedFeatures: string[], updatedAt }`（名前は任意・ローカルのみ、PIIなし）
- `sessionLog`: 直近1セッションの async/IOI ログのみ（検証・デバッグ用、件数上限あり）
- 増大時は IndexedDB へ移行可。

## 11. 非機能要件

- 対応: 最新 Chrome/Edge/Firefox/Safari（デスクトップ＋モバイル）。
- iOS Safari: `outputLatency` 欠如時 `baseLatency`、`getOutputTimestamp` 欠如時フォールバック。音声開始はユーザジェスチャ必須。
- オフライン: 初回ロード後ネット不要（音は合成・外部アセットなし）。PWA化は任意。
- 幼児UI: 大きいタップ標的、文字最小（プレリーダー前提）、音＋視覚の冗長提示、`touch-action: none`、`click`でなく`pointerdown`。
- 検証: デバッグモードで async ヒストグラム／SMT収束を可視化。
- 技術: Vite + TypeScript。エンジンはフレームワーク外の素TS。UIフレームワークを使う場合もゲームループ／音スケジューリングは描画サイクルに入れない。

### 11.1 配信・デプロイ（GitHub Pages）

- **静的配信でよい**: Vite ビルド成果物は素の HTML/CSS/JS。サーバ・バックエンド不要（永続化は localStorage）。GitHub Pages にそのまま載る。
- **ベースパス**: サブパス配信（`https://<user>.github.io/beat-buddy/`）のため `vite.config.ts` に **`base: '/beat-buddy/'`** を設定（確定）。未設定だとアセットが 404。カスタムドメイン/ユーザーサイトならルート配信で `base: '/'`。
- **HTTPS**: Pages は HTTPS 配信。Web Audio の安全コンテキスト要件を満たす。マイク非目標のため権限フローも不要。
- **アセット自己完結**: 音は合成、外部 CDN・音源ファイルなし。公開先・オフラインでも安定。相対パスで参照。
- **デプロイ**: GitHub Actions で `build` → `dist/` を Pages へ publish する定番ワークフロー。`main` への push でビルド・公開。
- **公開範囲**: Pages 公開はリポジトリが Private でも可。localStorage は訪問者ブラウザローカルのため利用者間でデータは混ざらない。
- **ルーティング**: SPA でクライアントルーティングを使う場合、Pages は 404 で SPA フォールバックしないため、ハッシュルーティングか `404.html` フォールバックを用意（単一画面遷移なら不要）。

## 12. チューニング可能パラメータ（試遊後に調整）

| 項目 | 既定値 | 根拠/備考 |
|---|---|---|
| 同期ヒット窓 | r≤0.15 / ≤0.30 | 就学前は広め・非罰。IBI比でテンポ自動スケール |
| こだまIOI許容 | ≤0.30 | 同上、緩め開始 |
| 知覚試行数 | 4問/セッション | 低負荷 |
| デバイス較正拍数 | 16拍（先頭2破棄） | 中央値の安定 |
| SMT較正上限 | 12タップ/8秒 | 早い方 |
| SMTデバウンス | ITI<120ms | 誤ダブルタップ |
| SMTクランプ | 300–700ms | 幼児SMTの妥当域 |
| テンポ段階 | ±10%/±20% | SMT基準 |
| 親ゲート機構 | 長押し＋一段階算数 | 幼児が解けず親は容易 |
